## 2. テーマ決定 ⭐

### ✅ 採用テーマ: **小集団 vs 大集団での意思決定の違い**

### 採用理由
- 構図がシンプルで参加者・観察者双方にわかりやすい
- 「社会構造・協力系」の論点を広く内包できる
- 以下の発展論点をすべて自然に包含する:
  - 男女比・国籍・年齢による思考の違い
  - リーダーの自然発生、忖度行動の発生
  - 議論の諦め・食い違いの発生
  - 規範・ルールの自発的生成と維持
  - 認知・デマ・意思決定プロセス
  - フリーライド(無言でやり過ごす人)の出現

### 棄却方向だった論点
- 経済的格差テーマ: 領域特有の専門性が必要でチューニングしにくい

## 3. 集団の定義

### 3.1 人数設定

| 集団 | 人数 | 備考 |
| --- | --- | --- |
| 小集団(スタートアップ) | **2〜10人(可変)** | 下限2人でも動く仕様にし、「どこから大集団化するか」を観察できるように |
| 大集団(大企業) | **100人** | 1000人は計算資源的に厳しいため100人に仮置き |

- **理由**: 人数=LLMの数 → 増やしすぎるとPCがパンクする制約あり
- 可変にすることで、**「何人から大集団化するか」の閾値自体を発見できる**可能性を重視

### 3.2 構造の整理
- 「大集団の中に小集団がある」構造(日本のレガシー企業のピラミッド型)も存在するが、今回は**シンプルに大企業 vs スタートアップの並列比較**で進める

## 4. エージェント属性の設定

| 属性 | 基本設定 | 備考 |
| --- | --- | --- |
| 男女比 | **8:2**(男性多め) | 時間があれば7:3も比較したい |
| 国籍 | 日本人ベース、**10人中1人程度外国人** | 外国人の人数閾値で創発が変わるか観察 |
| 年齢層 | 年齢のみ付与 | 年齢「だけ」与えて、役職等は設定しない |
| 役職 | **設定しない**(決めない) | 自然発生を観察するため |

### 設計ポリシー(兵頭氏アドバイス準拠)
- **指示を書きすぎない**。プロンプトで命令するほど創発が死ぬ
- 役職・階層は「勝手に握手が決まっていく」様を見たいので**あえて定義しない**
- LLMの国籍差(作られた国による性格差)も観察対象として興味深い

## 5. 3階層モデルの設計

兵頭氏の1階/2階/3階モデル([キックオフ議事録 §6.6](../../99_その他/2026-04/2026-04-22_LLMハッカソン_キックオフ議事録.md))に従って、本シミュレーションの各階層を定義。

### 5.1 1階(物) — 設定する
- **オフィス**(会議室を内包する前提)

### 5.2 2階(環境) — 一部のみ設定
| 項目 | 設定方針 | 理由 |
| --- | --- | --- |
| 景気(良い/悪い) | **軽く付与** | 全体に緩く効く前提条件として |
| 時間的プレッシャー | **あえて設定しない** | 創発として観察したい |
| 情報の流通量 | **あえて設定しない** | 創発として観察したい |
| 企業文化の雰囲気 | **あえて設定しない** | 大企業/スタートアップに固定化されてしまうため |

### 5.3 3階(世界の法則) — 選別して設定
「誰も逆らえない世界のルール」として、以下の**人間のキャパシティに相当するもの**を設定。

| 法則 | 設定 | 非設定 |
| --- | --- | --- |
| コミュニケーションの到達距離 | ✅ 設定する |  |
| 情報伝達の遅延 | ✅ 設定する |  |
| 認知限界(ダンバー数) | ✅ 設定する |  |
| 合意形成ルール(多数決/全会一致) |  | ❌ あえて設定しない(最も見たい論点なので) |
| 発言権の分配 |  | ❌ あえて設定しない(決めたらそうなってしまう) |

### 5.4 設計の核となる考え方
- **「あえて設定しない」箇所が説明変数**になる。振り返り時に「何が勝手に創発したか」を判定する材料として機能する
- LLMが「人間を超越」しないよう、人間のキャパシティ(記憶・認知)の範囲内に法則を収める

## 6. 環境の4象限と比較方式

### 6.1 4象限構造
「都心 vs 地方」× 「大企業 vs スタートアップ」= 4象限

|  | 地方 | 都心 |
| --- | --- | --- |
| 大企業 | □ | □ |
| スタートアップ | □ | □ |

### 6.2 進め方
- **まず片方の軸(例: 都心)で2つ(大企業/スタートアップ)を回す**
- 時間に余裕があれば、もう片方(地方)も実施
- エージェントのプロンプトは「あなたは◯◯(場所)の△△(規模)にいる□□歳です」程度に留める

### 6.3 比較の実装方式(3通り、要検討)
1. 2つの箱を**同時並行**でステップを回す
2. **個別**で回して比較する
3. 箱を**くっつけて**比較する

→ 実装上は**箱2つの並列比較方式**が基本。具体形は実装フェーズで決定。

## 7. 実行パラメータ

| 項目 | 設定 |
| --- | --- |
| ステップ数 | **100ステップ**程度(それ以上は時間がかかるため) |
| ステップ時間計測 | まず10ステップ回して、ざっくり10倍で推定 |
| LLMモデル | **Uncensored系(脱制約モデル)**を予定 |

### LLMモデル選定の思想(兵頭氏との議論由来)
- 通常のLLMは安全制約により「殴ってやろう」「こいつをクビにしたい」等の**人間らしい衝動**を出さない
- 賢いLLMほど「頭のいい会話」になりがちで、純粋で衝動的な人間の側面が見えない
- **リミッター解除状態のモデル**を使うことで、より人間らしい行動データを取得
- Ollama等で動かせるベースで選定予定
