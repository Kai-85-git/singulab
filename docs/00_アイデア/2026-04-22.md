# アイデア(2026-04-22)

LLMハッカソン向けのシミュレーション題材候補と、兵頭氏の **3階層フレームワーク**(1階=物 / 2階=環境 / 3階=物理・重力)への落とし込み。

> 参考: [2026-04-22_LLMハッカソン_キックオフ議事録.md](../99_その他/2026-04/2026-04-22_LLMハッカソン_キックオフ議事録.md) §6.6

---

## 共通フレームワーク

| 階層 | 役割 | 具体例 |
| --- | --- | --- |
| **1階:物** | 物質的・空間的オブジェクト。エージェントが触れる・ぶつかる・掴める対象 | 地形、建物、道具、他エージェントの身体 |
| **2階:環境** | 目に見えないが影響する量。センサーで知覚 | 温度、湿度、光、濃度、音、電磁場、評判 |
| **3階:物理・重力** | 世界を駆動する法則 | 重力、拡散、流体、反応速度、経済法則 |
| **エージェント** | LLMを搭載し、1〜3階の情報をセンサーで認識 → 思考 → 行動 → 世界にフィードバック | 細胞 / 虫 / 人間 / ウイルス 等 |

### 設計のコツ(議事録 §6.4, §7 より)
- **プロンプトで命令しすぎない**(例: 「いつでも怒ってください」は創発を殺す)
- ペルソナや役割(「あなたは上司です」「あなたは細胞Aです」)は設定してOK
- 環境はエージェントへのインプットなので設定可。ただし「こう反応しろ」と書かない
- 同一条件でも LLM の確率性で毎回結果は変わる。再現性は `temperature` 等で制御可能(ただし短期間では深追いしない)

## アイデア1: ウイルス

### 🎯 何を見たいか
- 宿主間感染の **ダイナミクス**(集団免疫・変異・収束)
- 「ウイルス側に意志があった場合」どのような戦略が創発するか(宿主を殺しすぎない寄生戦略 vs 暴走)

### 階層モデル

| 階層 | 内容 |
| --- | --- |
| **1階:物** | 宿主(人/動物/細胞)、ウイルス粒子、空間(部屋・街)、バリア(マスク・ワクチン抗体) |
| **2階:環境** | 気温・湿度・紫外線、宿主免疫レベル、人口密度、飛沫/エアロゾル濃度 |
| **3階:物理・重力** | 空気中の拡散(ブラウン運動)、受容体との結合確率、時間経過による変異率 |

### エージェント設計

- **ウイルス粒子エージェント**: ペルソナ=「増殖したい」「宿主を乗り換えたい」。変異を選ぶ際に LLM が「宿主を殺しすぎると自分も滅びる」と学習するか観察
- **宿主エージェント**: 症状に応じて外出を控える / マスク装着 / 医療アクセス

### 期待される創発
- 弱毒化戦略への自発的シフト
- スーパースプレッダー的個体の自然発生
- 宿主側の行動変容 → 感染曲線の形状変化

## アイデア2: 物質 / 細胞

### 🎯 何を見たいか
- 細胞レベルで**協調 → 組織形成 → 生命らしさ**が立ち上がるか
- ルールベース(走化性 if 文)では出せない「細胞の気持ち」的挙動
- 参考: 議事録 §8.3(細胞研究者の発表) / §8.4(創発 vs 中央制御の境界)

### 階層モデル

| 階層 | 内容 |
| --- | --- |
| **1階:物** | 細胞膜・核・オルガネラ、隣接細胞、ECM(細胞外マトリクス)、血管壁 |
| **2階:環境** | 栄養(グルコース・アミノ酸)濃度、酸素分圧、pH、シグナル分子(サイトカイン)、シェアストレス |
| **3階:物理・重力** | 浸透圧、拡散、電気化学勾配、機械的応力、反応速度 |

### エージェント設計

- **細胞エージェント**: ペルソナ=「生き残りたい」「分裂したい」「役割を担いたい」。役割の自己選択(幹細胞が血管細胞になる決断を LLM がする)
- センサー: 局所の栄養・酸素・隣接細胞の種類

### 期待される創発
- 血管の自己組織化(栄養勾配へのフィードバック)
- アポトーシス(利他的自死)が集団最適として出現
- 「過剰増殖=がん化」する細胞がルールでなく**意思決定**として現れるか

## アイデア3: 虫(昆虫コロニー)

### 🎯 何を見たいか
- 従来のルールベース群知能(蟻のフェロモン追従、蜂の8の字ダンス)と比較して、**LLM版は何が違うか**
- 「仕事をサボる個体」「危機時の利他行動」「集団離脱」など**非合理的な個の逸脱**が観察されるか

### 階層モデル

| 階層 | 内容 |
| --- | --- |
| **1階:物** | 巣、餌、天敵、植物、地形、仲間の身体 |
| **2階:環境** | フェロモン濃度、匂い、光量、温湿度、気象、音 |
| **3階:物理・重力** | 飛行空気力学、表面張力(水面歩行)、重力、消費エネルギー |

### エージェント設計

- **働き蟻/蜂エージェント**: ペルソナ=「疲れた」「怖い」「仲間を守りたい」を持つ。役割(採餌/保育/警備)は固定せず LLM が選択
- **女王/巣エージェント**(任意): 中央制御なしでも機能するか?

### 期待される創発
- エネルギー枯渇時の「仕事量低下」を個体差として観察
- 天敵出現時の「囮になる個体」の自発出現
- 情報(フェロモン)に嘘が混じった際の信頼ダイナミクス

## アイデア4: 人間(社会シミュレーション)

### 🎯 何を見たいか
- 都市スケールの **社会創発**(渋滞・流行・世論形成・派閥発生)
- ペルソナ(国籍・職業・性格)変更で社会全体の振る舞いがどう変わるか
- 議事録 §6.7 の渋谷シミュレーションの拡張版

### 階層モデル

| 階層 | 内容 |
| --- | --- |
| **1階:物** | 建物、道路、店舗、交通手段、他エージェントの身体、持ち物(スマホ・食料) |
| **2階:環境** | 気候、経済指標、情報(SNS・ニュース)、文化規範、評判(信頼スコア)、時間帯 |
| **3階:物理・重力** | 重力、移動速度の上限、エネルギー(空腹/疲労)、経済法則(需給) |

### エージェント設計

- **個人エージェント**: ペルソナ=年齢/職業/性格/出身/価値観。内面独白も出力(「上司がウザい」)
- **組織エージェント**(任意): 企業・政府が LLM 意思決定

### 期待される創発
- 経路選択の **非合理的多様性**(「気分で遠回り」)
- 情報源による **意見クラスタの分化**
- ペルソナに "火星帰り" を混ぜた場合の文化衝突

### サブアイデア
- **月面社会**(議事録 §8.1): ルナークラフト世界 + 資源制約 + 国際協力/競争
- **火星移住**(議事録 §8.2): Singulab Space の火星スレ議論をペルソナ化
- **アンケート調査 Mini** (§7 Q より): 5人のペルソナで商品選考、2ラウンドで収束

## アイデア5: ウイルス × 人間(組み合わせ)

複数アイデアの掛け算で、階層が自然にネストする例。

| 階層 | 内容 |
| --- | --- |
| **1階** | 人間の身体、ウイルス粒子、建物、マスク |
| **2階** | 気候、人口密度、ウイルス濃度、感染者の評判(差別) |
| **3階** | 空気拡散、感染確率、免疫減衰、情報伝播速度 |

- **人間エージェント**: 感染回避行動 + 経済活動 + 社会的つながり
- **ウイルスエージェント**: §アイデア1 と同じ

感染症 × 経済 × 差別 の 3軸で社会がどう揺らぐかを観察。

## 実装の現実解(議事録 §7 の制約を踏まえて)

| 観点 | 推奨 |
| --- | --- |
| エージェント数 | まず **5〜20体**。動作確認後に増やす |
| ステップ数 | 目的次第。アンケート型なら **2ラウンド**、社会型なら **50〜100ステップ** |
| LLM | **Ollama + 軽量モデル**(Llama 3 8B / Qwen 2.5 7B 等)で開始 |
| ハード | ゲーミングPC無くても、小規模なら MacBook Air でも可(議事録 §Q&A) |
| 再現性 | 同条件で 3〜10回回し、統計評価 |
| 提出優先度 | **動くこと > 規模** |

## 評価ポイント(議事録 §2, §7 より再掲)

1. **実装力**: 形になっているか
2. **世界観・ペルソナの設計**: プロンプトが命令調になっていないか
3. **創発性**: ルールで書けないことが起きているか
4. **社会実装への橋渡し**: アイデアがビジネス/研究にどう繋がるか
