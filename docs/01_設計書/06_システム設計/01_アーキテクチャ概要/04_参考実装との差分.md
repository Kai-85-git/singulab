# 04_参考実装からの変更方針(差分)

| 項目 | 参考実装 | 本システム(たたき案) | 理由 |
| --- | --- | --- | --- |
| 階層モデル | 暗黙(場所・火事のみ) | **1階/2階/3階を明示**し、プラグインで差し替え | 議事録 §6.6 のフレームワーク準拠 |
| イベント | 火事のみ | 汎用 `Event` 抽象を導入(火事・地震・物資補給・感染など) | 多様なシナリオに対応 |
| エージェントのペルソナ | 性別(male/female)のみ | ペルソナフィールドを **任意 dict** に拡張(年齢・職業・価値観等) | 社会シミュレーション対応 |
| センサー | place_status + fire_info(固定) | **環境量(2階)を統一 API で提供**(温度・濃度・評判等) | 1階/2階/3階の切り替えを容易に |
| 行動空間 | 4 方向 + stay | 4 方向 + stay を **基本**、ただしプラグインで拡張可能(例: 「話す」「交換」「攻撃」) | ハッカソン最小実装 → 拡張 |
| コミュニケーション制約 | 場所内/場所外の 2 領域 | 同仕様を維持(MVP)。将来的にはチャンネル(声/SNS/文書)を分ける | まず最小で動かす |
| LLM クライアント | Ollama 専用 | `LLMClient` インタフェース化し、API 系(Claude/GPT)も差し替え可能に | モデル比較検証のため |
| ログ | messages / memory_reasoning / frames | **イベントログ・創発指標ログ** を追加 | 嘘・派閥・通貨等の検出 |
