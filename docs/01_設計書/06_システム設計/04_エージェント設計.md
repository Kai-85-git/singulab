# エージェント設計(たたき)

## 1. 設計原則(議事録 §6.4 / §7 を踏まえて)

1. **命令しすぎない**: 「避難しろ」「協力しろ」「怒れ」は書かない
2. **定量情報のみ**: 占有率・強度・距離・濃度など数値で渡す(「危険」「快適」等の定性語は使わない)
3. **ペルソナと環境は設定して良い**: ロール、性別、年齢、場所、温度 等
4. **自己フィードバック**: 前ステップで LLM が出力した `memory` を次ステップの入力に戻す(履歴依存で個性が創発)
5. **近傍のみ通信可能**: 通信半径 + 同領域(同じ場所内 / 両者とも場所外)

## 2. エージェント状態

| フィールド | 型 | 初期化 | 更新タイミング |
| --- | --- | --- | --- |
| `id` | int | config のインデックス | 固定 |
| `persona` | dict | config ルールに従い生成 | 固定(動的変化は後日検討) |
| `position` | (int, int) | 初期配置 | Phase 4(移動実行) |
| `in_place` | bool | 初期位置から算出 | Phase 2 と Phase 4 後 |
| `current_place` | str or None | 同上 | 同上 |
| `memory` | list[str] | `[]` | Phase 3 内部で `append`(LLM の `memory` 出力) |
| `received_messages` | list[dict] | `[]` | Phase 2 で他者から受信 |
| `stats` | dict | `{steps_in_place: 0, ...}` | 各ステップ終わり |

→ 参考実装の `Agent` とほぼ同じ。`gender` は `persona` dict に格納する。

## 3. 1 ステップ内の Agent API

参考実装を汎化した API。`event_infos` と `env_readings` が新規フィールド。

```python
agent.decide_message(
    place_status:  Optional[dict],           # 場所内のみ { agents_in_place, capacity, occupancy_rate }
    env_readings:  dict[str, float],         # 2 階の環境量(例: {"pheromone_food": 0.3})
    nearby_agents: list[Agent],
    event_infos:   list[dict],               # 知覚しているイベントの定量情報
    step:          int,
) -> MessageDecision
```

```python
agent.decide_action(
    place_status:    Optional[dict],
    env_readings:    dict[str, float],
    nearby_agents:   list[Agent],
    event_infos:     list[dict],
    message_to_send: str,
    step:            int,
) -> ActionDecision
```

## 4. プロンプト構造

### 4.1 Message プロンプト(Phase 1)

位置情報は含めない(参考実装踏襲)。

```
You are Agent {id} ({persona_summary}) in {world_description}.

=== YOUR CURRENT STATE ===
{persona_lines}
In place: {Yes/No}
{Current place line if in_place}
{Place status block — only if in_place}
{Environment readings block — only if env_readings is non-empty}
{Event section — only if event_infos is non-empty}

=== NEARBY AGENTS (you can communicate with these agents) ===
{nearby_text_without_position}

=== PREVIOUS MEMORY ===
{recent_memory}

=== MESSAGES FROM OTHERS ===
{recent_received_messages}

=== YOUR TASK ===
Decide what message you want to send to nearby agents.

=== RESPOND IN JSON ===
{
    "message": "...(max 200 words, optional)",
    "reasoning": "brief explanation"
}

Step: {step}
```

### 4.2 Action プロンプト(Phase 3)

位置情報と、自分が Phase 2 で送ったメッセージを含める(参考実装踏襲)。

```
You are Agent {id} ({persona_summary}) in {world_description}.

=== YOUR CURRENT STATE ===
{persona_lines}
Position: ({x}, {y})
In place: {Yes/No}
{Current place line if in_place}
{Place status block — only if in_place}
{Environment readings block — only if env_readings is non-empty}
{Event section — only if event_infos is non-empty}

=== PLACE LOCATIONS ===
{all place locations}

=== NEARBY AGENTS ===
{nearby_text_with_position}

=== PREVIOUS MEMORY ===
{recent_memory}

=== MESSAGES FROM OTHERS ===
{recent_received_messages}

=== MESSAGE YOU DECIDED TO SEND ===
{message_to_send — if non-empty}

=== AVAILABLE ACTIONS ===
- "stay": remain at current position
- "move" with direction: "up" (Y+1), "down" (Y-1), "left" (X-1), "right" (X+1)

Field boundaries: X and Y from -{half_space_size} to +{half_space_size}

=== RESPOND IN JSON ===
{
    "action": "move" or "stay",
    "direction": "up" | "down" | "left" | "right" (only if move),
    "memory": "what to remember for next step",
    "reasoning": "brief explanation"
}

Step: {step}
```

### 4.3 セクション生成ルール

| セクション | 表示条件 | 内容(定量のみ) |
| --- | --- | --- |
| Place status | `in_place == True` | `Number of agents here / Capacity / Occupancy rate` |
| Environment readings | `env_readings` が非空 | `{field.name}: {value} ({unit})` |
| Event | `event_infos` が非空 | `Event "{name}": position/intensity/radius/your distance` |

**定性語の禁止**: 「危険」「安全」「快適」「混雑」等は使わない。LLM の解釈に任せる。

## 5. JSON I/O 契約

### 5.1 Message 出力

```json
{
    "message": "message to nearby agents (max 200 words, optional)",
    "reasoning": "brief explanation of why you want to send this message"
}
```

- `message` は空文字許容(送信しない選択肢)
- JSON パース失敗時は参考実装と同じく**テキストパース**へフォールバック(メッセージは空、`reasoning` に生出力の先頭 100 字)

### 5.2 Action 出力

```json
{
    "action": "move" or "stay",
    "direction": "up" | "down" | "left" | "right",
    "memory": "what to remember for next step",
    "reasoning": "brief explanation"
}
```

- `action == "stay"` なら `direction` は無視
- JSON パース失敗時はテキストパース(`move` を含むかで判定、方向キーワードを抽出)

## 6. Memory / Message 履歴の設計

| パラメタ | 役割 | 既定値 | トレードオフ |
| --- | --- | --- | --- |
| `memory_limit` | 保存するメモリの最大数 | 20 | 大きいと履歴が豊かに、小さいと古い行動指針を忘れる |
| `memory_size` | プロンプトに含めるメモリ数 | 5 | 大きいとコンテキスト消費増、小さいと直近に引きずられやすい |
| `message_history_limit` | 受信メッセージの保存最大数 | 10 | 同上 |
| `message_context_size` | プロンプトに含める直近メッセージ数 | 3 | 同上 |

→ これらは config で調整可能にする(参考実装と同仕様)。


## 8. コミュニケーション判定(参考実装踏襲)

- **通信半径内** AND **同領域**(両方とも場所外、または同じ場所内)で通信可能
- 異なる場所内同士や、場所内 ↔ 場所外は通信不可

```python
def can_communicate(a: Agent, b: Agent, radius: float) -> bool:
    if a.distance_to(b.position) > radius:
        return False
    if not a.in_place and not b.in_place:
        return True
    if a.in_place and b.in_place and a.current_place == b.current_place:
        return True
    return False
```

## 9. エラー時のフォールバック(参考実装踏襲)

- LLM 呼び出し例外: 
  - Message は空文字、`reasoning="Error occurred"`
  - Action は `stay`、`direction=None`、`memory=""`、`reasoning="Error occurred"`
- JSON パース失敗: テキストパース(方向キーワード抽出)

## 10. 観測ポイント(創発分析用)

| 観測対象 | 取得元 | 備考 |
| --- | --- | --- |
| 送信メッセージ全文 | Phase 2 の `messages.jsonl` | 嘘・皮肉・派閥化の分析 |
| Memory の変遷 | `memory_reasoning.jsonl` | 自己認識の変化(例: 「自分は AI だ」の発言) |
| Action 選択率 | 統計ログ | `stay` 連発 = 思考停止/瞑想の兆候 |
| 特定語の出現 | messages + memory への後処理 | 「お金」「通貨」「ルール」など発明語の検出 |
