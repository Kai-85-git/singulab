# アーキテクチャ概要(たたき)

## 1. 全体像

参考実装 [2d-multi-places-simulation-on-fire-public](../../10_共有資料/2d-multi-places-simulation-on-fire-public/) をベースに、以下の 5 要素で構成する。

```
┌─────────────────────────────────────────────────────────────┐
│                     Simulation Orchestrator                  │
│  (ステップ駆動 / イベント活性化 / 統計収集 / ログ書き出し)   │
└─────────┬─────────────┬─────────────┬───────────────┬───────┘
          │             │             │               │
          ▼             ▼             ▼               ▼
    ┌──────────┐  ┌───────────┐ ┌────────────┐ ┌────────────┐
    │  World   │  │  Agents   │ │   Events   │ │  Visualizer│
    │ (場・場所) │  │ (LLM搭載) │ │(火事/災害等)│ │(描画/統計)  │
    └────┬─────┘  └─────┬─────┘ └──────┬─────┘ └────────────┘
         │              │               │
         └──────────────┼───────────────┘
                        ▼
                  ┌───────────┐
                  │ LLM Client │
                  │  (Ollama)  │
                  └───────────┘
```

## 2. コンポーネント責務

| コンポーネント | 責務 | 参考実装の対応 |
| --- | --- | --- |
| **Simulation Orchestrator** | ステップ駆動、Phase 1〜4 の順序制御、イベント活性化、統計収集、ログ書き出し | `simulation.py` |
| **World(場・場所)** | 座標系、場所(place)ごとの `center / half_size / capacity`、場所内判定、通信可能判定 | `utils.py`(`PlaceConfig`, `is_position_in_place`)|
| **Agent** | LLM への推論依頼、Memory / Message / Action の生成と更新、近傍エージェント抽出 | `agent.py` |
| **Events(火事/他)** | 発生ステップ・中心位置・知覚半径・強度を持ち、エージェントに定量情報のみ提供 | `simulation.py` の `fire_states` 周辺 |
| **LLM Client** | Ollama API へのリクエスト、接続/モデル確認、生成オプション管理 | `ollama_client.py` |
| **Visualizer** | フレーム保存、統計プロット、カラーバー・通信半径・火事半径の描画 | `visualization.py` |

## 3. データフロー(1 ステップ)

```
(1) Orchestrator: イベント活性化判定(例: fire.start_step に達したら fire_state を生成)
(2) Orchestrator: 全 Agent の state 更新(in_place / current_place)
(3) Agent.decide_message() ← place_status / nearby_agents / fire_info
(4) Orchestrator: メッセージ配信(通信ルール適用)
(5) Agent.decide_action() ← place_status / nearby_agents / fire_info / 自分が送ったメッセージ
(6) Orchestrator: 移動実行 → state 更新 → 統計収集 → ログ書き込み
```

## 4. 参考実装からの変更方針(差分)

| 項目 | 参考実装 | 本システム(たたき案) | 理由 |
| --- | --- | --- | --- |
| 階層モデル | 暗黙(場所・火事のみ) | **1階/2階/3階を明示**し、プラグインで差し替え | 議事録 §6.6 のフレームワーク準拠 |
| イベント | 火事のみ | 汎用 `Event` 抽象を導入(火事・地震・物資補給・感染など) | 多様なシナリオに対応 |
| エージェントのペルソナ | 性別(male/female)のみ | ペルソナフィールドを **任意 dict** に拡張(年齢・職業・価値観等) | 社会シミュレーション対応 |
| センサー | place_status + fire_info(固定) | **環境量(2階)を統一 API で提供**(温度・濃度・評判等) | 1階/2階/3階の切り替えを容易に |
| 行動空間 | 4 方向 + stay | 4 方向 + stay を **基本**、ただしプラグインで拡張可能(例: 「話す」「交換」「攻撃」) | ハッカソン最小実装 → 拡張 |
| コミュニケーション制約 | 場所内/場所外の 2 領域 | 同仕様を維持(MVP)。将来的にはチャンネル(声/SNS/文書)を分ける | まず最小で動かす |
| LLM クライアント | Ollama 専用 | `LLMClient` インタフェース化し、API 系(Claude/GPT)も差し替え可能に | モデル比較検証のため |
| ログ | messages / memory_reasoning / frames | **イベントログ・創発指標ログ** を追加 | 嘘・派閥・通貨等の検出 |

## 5. 非機能要件(たたき)

| 項目 | 指針 |
| --- | --- |
| 性能 | エージェント数 **5〜20** 体で 100 ステップを数分以内。LLM 呼び出しは逐次で可(並列化は後段) |
| 再現性 | `seed` を config に持たせる。LLM は `temperature` を明示。3〜10 回試行で統計比較 |
| 可観測性 | すべての LLM 入出力を `*.jsonl` で残す(後で解析可能にする) |
| モジュール性 | シナリオ追加時に `simulation.py` を大改修しなくても済むよう、World/Event/Agent の境界を明確に |
| 実行環境 | Python 3.10+ / Ollama(ローカル)+ 任意で API LLM。Windows / macOS で動かす |

## 6. スコープ外(MVP では扱わない)

- 分散並列実行(複数 PC で 1 シミュレーション)
- 完全なセーブ/リロード(途中復帰)
- 3D 空間
- 大規模エージェント(100 体超)
- リアルタイム対話 UI(まずはバッチ実行 + 事後の viewer)

## 7. 想定ユースケース別の構成例

| シナリオ | 1階 | 2階 | 3階 | 追加イベント |
| --- | --- | --- | --- | --- |
| **バー混雑 + 火事**(参考実装) | バー × 2 | 占有率・火事情報 | 火事強度 | Fire |
| **細胞の自己組織化** | 細胞・ECM | 栄養・酸素・サイトカイン | 拡散・反応速度 | Nutrient pulse |
| **ウイルス感染** | 人間・ウイルス粒子 | 飛沫濃度・免疫レベル | 拡散・結合確率 | Outbreak |
| **社会(市場)** | 店舗・商品・人間 | 価格・評判・噂 | 経済法則 | 需給変動 |
| **月面/火星独立運動** | 建物・通信機・物資 | 情報(地球からの指令)・経済指標 | 重力・エネルギー | 通信遮断 |

→ 各シナリオは **config と階層プラグインの差し替えだけで** 実現することを目標にする。
