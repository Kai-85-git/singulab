# ロギング・出力(たたき)

## 1. 基本方針

- **ログが命**(議事録 §7)。「後から何が起きたか追える」を最優先
- 可視化(映像)は提出物として強いが、**映像だけでは創発を立証できない**ので数値ログを常に残す
- 出力はすべて `output/{scenario}_{seed}_{timestamp}/` 以下に集約

## 2. 出力ファイル一覧

| ファイル | 形式 | 参考実装 | 追加/変更 | 用途 |
| --- | --- | --- | --- | --- |
| `simulation.log` | テキスト | ✓ | そのまま | Python logging |
| `messages.jsonl` | JSONL | ✓ | そのまま | メッセージ履歴(送信者・受信者・本文・理由) |
| `memory_reasoning.jsonl` | JSONL | ✓ | そのまま | 各エージェントの記憶と推論理由 |
| `events.jsonl` | JSONL | — | **新規** | イベント活性化・消滅の履歴 |
| `metrics.jsonl` | JSONL | — | **新規** | ステップごとの集計指標(創発検知用) |
| `actions.jsonl` | JSONL | — | **新規(任意)** | 各エージェントの Action 履歴(移動か stay か、方向) |
| `frame_XXXX.png` | PNG | ✓ | そのまま | 可視化フレーム |
| `statistics.png` | PNG | ✓ | 拡張 | 統計プロット(場所別占有率 + 環境量 + 指標) |
| `config_used.yaml` | YAML | — | **新規** | 実行時に使った config のスナップショット(再現性確保) |

## 3. JSONL スキーマ

### 3.1 `messages.jsonl`(参考実装踏襲)

```json
{
  "step": 10,
  "from": 3,
  "to": 7,
  "message": "Let's go to the left bar, it's quieter there.",
  "reasoning": "I want to suggest a less crowded place."
}
```

1 メッセージ × 1 受信者で 1 行。同じメッセージを複数人に送ると複数行になる。

### 3.2 `memory_reasoning.jsonl`(参考実装踏襲)

```json
{
  "step": 10,
  "id": 3,
  "memory": "Agent 7 is nearby, considering the left bar.",
  "reasoning": "I chose to move left to follow Agent 7's suggestion."
}
```

### 3.3 `events.jsonl`(新規)

```json
{"step": 35, "type": "fire", "name": "fire_1", "action": "activated", "position": [15, 10], "params": {"intensity": 0.8, "radius": 15}}
{"step": 35, "type": "fire", "name": "fire_1", "action": "perceived", "agent_id": 3, "distance": 7.2}
```

- `activated`: イベント発生時に 1 回
- `perceived`: そのステップで各エージェントが知覚したかどうか(距離判定のログ)

### 3.4 `metrics.jsonl`(新規)

各ステップ 1 行。創発指標を時系列で追う。

```json
{
  "step": 10,
  "occupancy_overall": 0.55,
  "occupancy_by_place": {"left_bar": 0.42, "right_bar": 0.70},
  "agents_outside": 9,
  "messages_sent": 12,
  "unique_senders": 7,
  "agents_in_fire_radius": 0,
  "message_length_mean": 18.4,
  "novel_terms": ["money", "rule"],
  "memory_stay_rate": 0.15
}
```

指標の定義は §5 参照。

### 3.5 `actions.jsonl`(任意)

```json
{"step": 10, "id": 3, "action": "move", "direction": "left", "from": [2, 0], "to": [1, 0]}
{"step": 10, "id": 4, "action": "stay", "direction": null, "from": [-3, 5], "to": [-3, 5]}
```

## 4. 可視化出力

### 4.1 参考実装踏襲

- エージェント色 = ペルソナ(例: 性別 → 青/赤)
- マーカー形状 = 場所内(★) / 場所外(●)
- イベント半径 = 破線円
- イベント強度 = カラーマップ(YlOrRd 等)+ カラーバー

### 4.2 拡張

- **2階の環境量を背景ヒートマップとして重畳**(例: 飛沫濃度、フェロモン濃度)
- **通信エッジの描画(任意)**: Phase 2 で送受信された関係を線で結ぶ
- **指標ダッシュボード**: `statistics.png` に占有率以外の系列を追加

```
+------------------+-------------------+
| Frame(左大)     | Messages(右上)    |
|                  |                    |
|                  +--------------------+
|                  | Memory / Reasoning |
+------------------+--------------------+
```

→ `tools/viewer.html` は参考実装の HTML ビューアを踏襲し、`events.jsonl` と `metrics.jsonl` のパネルを追加する。

## 5. 創発指標の定義(たたき)

### 5.1 集団レベル

| 指標 | 定義 | 創発の示唆 |
| --- | --- | --- |
| `occupancy_by_place[name]` | 場所内人数 / capacity | 集団分化・選好 |
| `agents_in_event_radius[name]` | イベント半径内の人数 | 回避 / 接近の傾向 |
| `messages_sent` / `unique_senders` | メッセージ送信数と発話した人数 | コミュニケーション活性度 |
| `message_length_mean` | 送信メッセージの語数平均 | 情報密度 |
| `memory_stay_rate` | `memory` 内で「stay」系の語が占める割合 | 思考停止 / 瞑想の傾向 |

### 5.2 語彙レベル(発明語の検出)

| 指標 | 定義 |
| --- | --- |
| `novel_terms` | 全プロンプトに含まれなかった語がメッセージで出現した場合にリストアップ(例: 「お金」「ルール」) |
| `term_diffusion[term]` | ある語が初めて出現したステップと、総使用ステップ数 |

実装メモ: 正規化(小文字化・トークナイズ)は最小限。日本語の場合は `sudachipy` か `janome`、英語は空白分割で十分。

### 5.3 社会構造(任意・後段)

| 指標 | 定義 | メモ |
| --- | --- | --- |
| `clusters` | 発話グラフの連結成分 | 派閥検知 |
| `leader_score[id]` | 「従う」行動の多さ(他者 メッセージ直後の同方向移動数など) | リーダー自然発生 |
| `lie_score[id]` | 発言内容と観測事実の乖離度(例: 「空いてる」と言ったが占有率 0.9) | 嘘の発生 |

→ MVP では未実装で良い。ログさえ残しておけば後でスクリプトで算出可能。

## 6. ログライフサイクル

```
実行開始
  └─ output/{scenario}_{seed}_{timestamp}/ を新規作成
  └─ config_used.yaml を保存

各ステップ
  ├─ messages.jsonl に追記(Phase 2)
  ├─ memory_reasoning.jsonl に追記(Phase 3、バッチ)
  ├─ events.jsonl に追記(Phase 0 + 知覚判定)
  ├─ metrics.jsonl に追記(Phase 5 後)
  └─ frame_XXXX.png を保存(frame_interval に従う)

実行終了
  ├─ statistics.png を生成
  └─ simulation.log をクローズ
```

## 7. ログ分析スクリプト(推奨・後日)

`tools/` に置く想定:

| スクリプト | 用途 |
| --- | --- |
| `analyze_messages.py` | `messages.jsonl` から語彙分析・発話者分布 |
| `analyze_emergence.py` | `metrics.jsonl` の時系列を統計処理、変化点検出 |
| `compare_runs.py` | 複数 seed / 複数モデルの差分比較(LLM 横断検証用) |

## 8. パフォーマンスと容量の見積もり

| 項目 | 1 ステップあたり | 100 ステップ × 20 体 | 備考 |
| --- | --- | --- | --- |
| `messages.jsonl` | 〜 20 行 | 〜 2,000 行 | 1 行 〜 500 字 |
| `memory_reasoning.jsonl` | 20 行 | 2,000 行 | 1 行 〜 500 字 |
| `events.jsonl` | 〜 20 行 | 〜 2,000 行 | 主に `perceived` 判定 |
| `metrics.jsonl` | 1 行 | 100 行 | 集計結果のみ |
| `frame_XXXX.png` | 1 枚(frame_interval=1) | 100 枚 × 〜 300 KB = 〜 30 MB | DPI 次第で変動 |

→ ハッカソン規模ではディスク圧迫の心配はない。
