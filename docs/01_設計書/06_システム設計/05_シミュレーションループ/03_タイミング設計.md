# 03_タイミング設計の要点

## 1. なぜ「Message → Action」の順か(参考実装踏襲)

- 先にメッセージを決めることで、「発話内容を踏まえた行動」が可能になる
- 例: 「逃げよう」と言った → 自分も動く / 「ここ安全」と言った → 自分は残る
- 逆順(Action 先)だと、発話が行動と整合しにくくなる

## 2. 位置情報の露出制御

| Phase | 位置情報 | 理由 |
| --- | --- | --- |
| Phase 1(Message) | **非公開** | 位置をメッセージに直接書かせない(情報伝達を創発させるため) |
| Phase 3(Action) | **公開** | 移動判断に必要 |

## 3. イベント知覚の一貫性

- Phase 1 と Phase 3 の両方で、同一の `event_infos` を渡す
- つまり「メッセージは書くが、行動判断では知らない」のような不整合は発生しない
