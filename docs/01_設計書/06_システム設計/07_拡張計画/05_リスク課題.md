# 05_リスク・課題(たたきの時点で認識しているもの)

| リスク | 影響 | 対策(候補) |
| --- | --- | --- |
| LLM 呼び出しコスト(時間) | 100 ステップ × 20 体 × 2 呼び出し = 4000 回 | Phase 内並列化(asyncio) / 小型モデル優先 / キャッシュ |
| 創発が発生しない | ハッカソン成果が弱い | temperature 調整・Uncensored モデル試行・プロンプト最小化 |
| 過剰設計で実装が間に合わない | スケジュール遅延 | M0〜M1 までを最優先。M2 以降は余力次第 |
| 出力 JSON のパース失敗 | エージェントが `stay` 連発 | モデルごとの JSON 追従力を事前検証 / フォールバックをこまめに |
| 参考実装のライセンス(GPLv3) | 派生物も GPLv3 に縛られる | 提出物のライセンス表記を整理、必要ならクリーンルーム実装を検討 |
| 日本語 vs 英語プロンプト | モデルによっては片言に | 日本語強いモデル(Swallow / Sarashina / Aya)と併用検証 |
