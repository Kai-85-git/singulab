## 2. モジュール責務表

### 2.1 Orchestrator(`src/simulation.py`)

- config の読み込み
- World / Agents / Events / LLMClient / Visualizer の初期化
- ステップ駆動(`step_simulation()`)とイベント活性化
- 統計収集・ログ書き出し
- 実行終了後の統計サマリ算出

### 2.2 World(`src/world/`)

| ファイル | 責務 |
| --- | --- |
| `place.py` | 場所(`Place`)の定義。`name / type / center / half_size / capacity` |
| `field.py` | フィールド全体の座標系と境界処理、位置 → 場所の解決(`get_place_at_position`)|
| `environment.py` | 2階: 環境量(温度・濃度・評判 等)を位置から取得する API を提供 |

### 2.3 Physics(`src/physics/`)

- 3階: 環境量の時間発展(拡散・減衰・反応)を担う
- `base.py` に `Physics.step(env, dt)` 抽象を置く
- MVP では `diffusion.py` の単純なガウス拡散のみ実装
- 重力・経済法則は差し替え可能なプラグインとして後から追加

### 2.4 Events(`src/events/`)

- `base.py` に `Event` 抽象
  - `activate(sim, step)`: 発生判定
  - `perceived_by(agent)`: そのエージェントが知覚する定量情報を返す(Noneなら非知覚)
  - `is_active(step)`: 継続中か
- `fire.py` は参考実装の挙動を踏襲(時間不変・距離依存知覚)
- 将来的に `earthquake.py`, `supply.py`, `infection.py` 等を追加

### 2.5 Agent(`src/agent/`)

| ファイル | 責務 |
| --- | --- |
| `agent.py` | 状態(位置・in_place・メモリ)管理、行動実行、近傍エージェント取得 |
| `persona.py` | ペルソナ辞書(参考実装の `gender` を汎化: `{"gender": "...", "age": 30, "role": "barista"}`) |
| `memory.py` | `memory_limit / memory_size` の循環バッファ、受信メッセージの履歴管理 |
| `prompt.py` | Message プロンプトと Action プロンプトの組み立て、出力 JSON のパース |

### 2.6 LLM(`src/llm/`)

- `base.py`: `class LLMClient: def generate(prompt: str) -> str`
- `ollama.py`: 参考実装の `OllamaClient` をそのまま移植
- `anthropic.py` / `openai.py` は **任意**(モデル比較検証用、MVP では不要)

### 2.7 Logging(`src/logging/`)

- `jsonl.py`: 参考実装の `messages.jsonl` / `memory_reasoning.jsonl` に加え、`events.jsonl`(イベント発生・知覚)を追加
- `metrics.py`: 創発指標(後述の `06_ロギング・出力.md` を参照)

### 2.8 Visualizer(`src/viz/`)

- 参考実装の可視化(エージェント色=性別、場所内外=マーカー形状、火事半径=破線円、カラーバー)を踏襲
- 拡張ポイントとして 2階の環境量をヒートマップで重畳できるようにする(例: 飛沫濃度)

## 4. インタフェース案(擬似コード)

```python
class Event(Protocol):
    name: str
    def activate(self, sim: "Simulation", step: int) -> None: ...
    def is_active(self, step: int) -> bool: ...
    def perceived_by(self, agent: "Agent") -> Optional[dict]: ...

class LLMClient(Protocol):
    model: str
    def generate(self, prompt: str) -> str: ...
    def check_connection(self) -> bool: ...

class Physics(Protocol):
    def step(self, env: "Environment", dt: float) -> None: ...

class Agent:
    def decide_message(
        self,
        place_status: Optional[dict],
        env_readings: dict,
        nearby_agents: list["Agent"],
        event_infos: list[dict],
        step: int,
    ) -> MessageDecision: ...

    def decide_action(
        self,
        place_status: Optional[dict],
        env_readings: dict,
        nearby_agents: list["Agent"],
        event_infos: list[dict],
        message_to_send: str,
        step: int,
    ) -> ActionDecision: ...
```

→ 参考実装の `fire_info` は `event_infos: list[dict]` に汎化した。`env_readings` は 2階の環境量を渡すための新規フィールド。

## 5. 最小実装順序(実装タスク粒度)

1. `src/world/` と `src/llm/ollama.py` を参考実装から切り出し
2. `src/agent/` を切り出し、`event_infos` 対応に軽くリファクタ
3. `src/events/fire.py` を `Event` 抽象下に再配置
4. `src/simulation.py` でオーケストレーション(参考実装の `Simulation` を整理)
5. `src/logging/jsonl.py` で既存 JSONL ロガーを移植 + `events.jsonl` を追加
6. ここまでで **参考実装と等価なバー + 火事シナリオ** を動かす(デグレ確認)
7. `src/world/environment.py` と `src/physics/diffusion.py` を追加し、細胞シナリオの最小版を作る
