# 設定スキーマ(たたき)

## 1. 設計方針

- **1 シナリオ = 1 YAML** を基本とし、`config/base.yaml` を include するかたちでデフォルト値を共有する
- 参考実装の `config.yaml` を**上位互換**で拡張する(既存の `places` / `fires` / `agents` / `llm` はそのまま動くこと)
- 1階(物)/2階(環境)/3階(物理)に対応するセクションを新設する
- 再現性のため `seed` を必須にする

## 2. トップレベル構造

```yaml
meta:
  scenario: "bar_fire"        # シナリオ名(ログ整理用)
  seed: 42                    # 乱数シード(必須)
  description: "参考実装と等価なバー + 火事シナリオ"

simulation:
  duration: 100               # ステップ数
  half_space_size: 25         # フィールド: -25 〜 +25
  dt: 1.0                     # 1 ステップあたりの時間刻み(3 階の物理で使う)

# 1階: 物(場所・障害物)
places:
  - name: "left_bar"
    type: "bar"
    center_x: -15
    center_y: 0
    half_size: 5
    capacity: 12
  - name: "right_bar"
    type: "bar"
    center_x: 15
    center_y: 0
    half_size: 5
    capacity: 10

# 2階: 環境(量)
environment:
  fields: []                  # MVP では空。将来: temperature / pheromone 等
  # 例:
  #   - name: "pheromone_food"
  #     initial: 0.0
  #     unit: "arbitrary"
  #     decay_rate: 0.01

# 3階: 物理法則
physics:
  laws: []                    # MVP では空。将来: diffusion / gravity 等
  # 例:
  #   - type: "diffusion"
  #     target: "pheromone_food"
  #     coefficient: 0.2

# イベント(参考実装の fires を汎化)
events:
  - type: "fire"
    name: "fire_1"
    start_step: 35
    params:
      intensity: 0.8
      radius: 15
      center_x: 15
      center_y: 10
  - type: "fire"
    name: "fire_2"
    start_step: 70
    params:
      intensity: 0.5
      radius: 8
      center_x: -10
      center_y: -10

# エージェント
agents:
  num_agents: 20
  communication_radius: 5
  memory_limit: 20
  memory_size: 5
  message_history_limit: 10
  message_context_size: 3

  # ペルソナ定義(参考実装の gender を包含)
  persona:
    fields:
      - name: "gender"
        values: ["male", "female"]
        assignment: "random"    # random / uniform / list
      # 例: 年齢・職業を追加する場合
      # - name: "age"
      #   range: [20, 60]
      #   assignment: "uniform"
      # - name: "role"
      #   values: ["customer", "bartender"]
      #   assignment: "list"

  # 初期配置ルール
  initial_placement:
    avoid_places: true
    distribution: "uniform"     # uniform / clustered

# LLM
llm:
  provider: "ollama"            # ollama / anthropic / openai
  model: "llama3.1:8b"
  base_url: "http://localhost:11434"
  temperature: 0.7
  max_tokens: 2048
  repeat_penalty: 1.1
  repeat_last_n: 128
  min_p: 0.05

# 可視化
visualization:
  save_frames: true
  frame_interval: 1
  output_dir: "output"
  overlays:
    env_heatmap: null           # 2 階フィールド名を指定するとヒートマップ化
    show_communication_radius: true
    show_event_radius: true

# ログ
logging:
  level: "INFO"
  log_file: "simulation.log"
  outputs:
    messages_jsonl: true
    memory_reasoning_jsonl: true
    events_jsonl: true
    metrics_jsonl: true         # 創発指標(後述)
```

## 3. 主要セクションの説明

### 3.1 `meta`

| キー | 型 | 必須 | 説明 |
| --- | --- | --- | --- |
| `scenario` | str | ○ | ログフォルダ名・リネーム用の識別子 |
| `seed` | int | ○ | 乱数シード(再現性確保) |
| `description` | str | | 人間向けメモ |

### 3.2 `places`(1 階)

参考実装と同一スキーマを踏襲。各場所は `name / type / center_x / center_y / half_size / capacity` 必須。

### 3.3 `environment`(2 階) — 新規

`fields` はリスト。各フィールドは以下:

| キー | 型 | 説明 |
| --- | --- | --- |
| `name` | str | 参照名(例: `pheromone_food`) |
| `initial` | float / ndarray | 初期値(スカラー or グリッド) |
| `unit` | str | 単位(プロンプトへの表示・解釈用) |
| `decay_rate` | float | 1 ステップあたりの減衰率(任意) |
| `source` | dict | 発生源(任意、例: `{position: [0,0], rate: 0.1}`) |

### 3.4 `physics`(3 階) — 新規

`laws` はリスト。各法則は `type` に応じてパラメタが変わる。

| type | パラメタ例 | 対象 |
| --- | --- | --- |
| `diffusion` | `target, coefficient` | 環境量の拡散 |
| `gravity` | `g, direction` | (将来) 粒子の落下 |
| `decay` | `target, half_life` | 環境量の減衰 |
| `reaction` | `reactants, products, rate` | 化学反応 |

### 3.5 `events` — 参考実装の `fires` を汎化

参考実装:
```yaml
fires:
  - name: "fire_1"
    start_step: 15
    intensity: 0.8
    radius: 10
```

本システム:
```yaml
events:
  - type: "fire"              # ← 汎化
    name: "fire_1"
    start_step: 15
    params:
      intensity: 0.8
      radius: 10
```

**互換読み込み**: 旧形式の `fires:` キーが来たら内部で `events:` + `type: fire` に変換する。

### 3.6 `agents.persona` — 新規

参考実装の「男/女」を包含する汎化。`fields` 配下に任意数のペルソナ属性を宣言できる。`assignment` で割当ルールを指定。

| assignment | 挙動 |
| --- | --- |
| `random` | `values` からランダム |
| `uniform` | `range` の一様乱数 |
| `list` | エージェント ID 順に `values` をラウンドロビン |

### 3.7 `llm.provider`

参考実装は Ollama 固定。本システムは `provider` キーで切り替え可能にする。MVP は `ollama` のみ。

| provider | 追加フィールド |
| --- | --- |
| `ollama` | `base_url` |
| `anthropic` | `api_key_env`(環境変数名) |
| `openai` | `api_key_env` |

## 4. バリデーション方針

- `seed` 未指定はエラー
- `places[].capacity <= 0` はエラー
- `events[].type` が未登録の場合はエラー(プラグイン側で型を登録する)
- `agents.num_agents > フィールド総セル数` はエラー(初期配置が不可能)

## 5. ディレクトリ構成(`config/`)

```
config/
├── base.yaml                      # 共通デフォルト(llm / logging / visualization)
├── scenario_bar_fire.yaml         # 参考実装と等価
├── scenario_cells.yaml            # 細胞シナリオ
├── scenario_virus_human.yaml      # ウイルス × 人間
└── scenario_moon_independence.yaml # 月面独立運動
```

CLI 側で `--config config/scenario_bar_fire.yaml` のように指定する。`base.yaml` はインクルード的に扱う(内部でマージ)。

## 6. マイグレーション・メモ

- 参考実装の `config.yaml` をそのまま読めるようにする「互換シム」を同梱する(最初の 1 週間)
- その後、`scenario_bar_fire.yaml` に正式移行。旧形式は deprecated 警告を出す。
