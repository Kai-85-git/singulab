# 03_3階層モデル設計

1階(物) / 2階(環境) / 3階(世界の法則) の 3 階層に分けて、設定する項目と「あえて設定しない」項目を選別する。

> 3 階層フレームワーク自体の出典: [キックオフ議事録 §6.6](../../../99_その他/2026-04/2026-04-22_LLMハッカソン_キックオフ議事録.md)

## 1. 1階(物) — 設定する

- **オフィス**(会議室を内包する前提)

## 2. 2階(環境) — 一部のみ設定

| 項目 | 設定方針 | 理由 |
| --- | --- | --- |
| 景気(良い/悪い) | **軽く付与** | 全体に緩く効く前提条件として |
| 時間的プレッシャー | **設定しない** | 創発として観察したい |
| 情報の流通量 | **設定しない** | 創発として観察したい |
| 企業文化の雰囲気 | **設定しない** | 大企業/スタートアップに固定化されてしまうため |

## 3. 3階(世界の法則) — 選別して設定

「誰も逆らえない世界のルール」として、**人間のキャパシティに相当するもの** を設定する。

| 法則 | 設定 | 備考 |
| --- | --- | --- |
| コミュニケーションの到達距離 | ✅ 設定する | 物理的・空間的制約 |
| 情報伝達の遅延 | ✅ 設定する | リアルタイム性の限界 |
| 認知限界(ダンバー数) | ✅ 設定する | 人間の記憶・認識の制約 |
| 合意形成ルール(多数決/全会一致) | ❌ **設定しない** | **最も観察したい論点のため** |
| 発言権の分配 | ❌ **設定しない** | 決めたらそのとおりになってしまう |

## 4. 設計の核

> 「あえて設定しない」箇所が説明変数になる。振り返り時に「何が勝手に創発したか」を判定する材料として機能する。LLM が「人間を超越」しないよう、人間のキャパシティ(記憶・認知)の範囲内に法則を収める。
