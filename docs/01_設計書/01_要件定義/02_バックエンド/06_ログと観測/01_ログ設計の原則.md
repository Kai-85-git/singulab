# 01_ログ設計の原則

1. **全て jsonl**(1 行 1 レコード、tail 可能、pandas で読める)
2. **前段処理しない**: raw で残す。加工は事後に pandas/DuckDB でやる
3. **ファイル分割**: 用途ごとに別ファイル(llm_io / perf / vram / events / emergence)
4. **timestamp は ISO8601 + epoch 両方**: 可読性と計算しやすさを両立
5. **run_id を全レコードに付与**: 複数実験を混在解析できるように
