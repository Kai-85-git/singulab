# 02_ディレクトリ構成

```
logs/
  runs/
    {run_id}/
      run.yaml            # 実験 metadata(モデル、seed、params、人数等)
      llm_io.jsonl        # 全 LLM 呼び出しの入出力
      perf.jsonl          # tokens/sec 等のパフォーマンス指標
      vram.jsonl          # VRAM 使用量のサンプリング
      events.jsonl        # シミュレーションのイベント(移動、会議開始等)
      emergence.jsonl     # 創発指標(合意、派閥、リーダー)
      transcripts/
        agent_{id}.jsonl  # エージェント別のタイムライン
```

## ER 図(jsonl 間の関係)

```mermaid
erDiagram
    RUN ||--o{ LLM_IO : has
    RUN ||--o{ PERF : has
    RUN ||--o{ VRAM : has
    RUN ||--o{ EVENTS : has
    RUN ||--o{ EMERGENCE : has
    RUN ||--o{ TRANSCRIPT : has
    RUN {
        string run_id PK
        string timestamp
        json models
        int agents
        int steps
    }
    LLM_IO {
        string run_id FK
        int step
        int agent_id
        string prompt_system
        string prompt_user
        json response_parsed
        int latency_ms
    }
    PERF {
        string run_id FK
        int step
        float throughput_tokens_per_sec
        int errors
    }
    VRAM {
        string run_id FK
        float ts_epoch
        int vram_used_mb
        int gpu_util_pct
    }
    EVENTS {
        string run_id FK
        int step
        string event_type
    }
    EMERGENCE {
        string run_id FK
        int step
        float gini
        int faction_count_est
    }
    TRANSCRIPT {
        string run_id FK
        int agent_id
        json timeline
    }
```

`run_id` をキーにすべての jsonl が結びつく。事後解析では `run.yaml` の metadata と組み合わせて分析する。
