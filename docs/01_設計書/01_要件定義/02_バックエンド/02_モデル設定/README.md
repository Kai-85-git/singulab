## 共通の推論パラメータ既定値

| パラメータ | 値 | 意図 |
| --- | --- | --- |
| `temperature` | 0.7〜0.9(創発観察)/ 0.2〜0.4(再現実験) | 議事録 §Q&A、温度を振って観察 |
| `num_ctx` | 2048 | VRAM 節約、本件は短文で十分 |
| `num_predict` | 256 | 1 発話を短文に制限 |
| `top_p` | 0.9 | デフォルト |
| `repeat_penalty` | 1.1 | 繰り返し抑制 |
| `seed` | 固定必須 | 再現性 |

→ モデルごとの推奨値は子ファイルを参照。

## SYSTEM プロンプト設計方針(全モデル共通)

### やること
- 状態記述のみ(「あなたは ◯◯ です」)
- ペルソナ属性(年齢・性別・国籍・場所・規模)
- 環境状態(景気)

### やらないこと
- ❌ 行動の指示(「丁寧に話してください」「議論してください」)
- ❌ 役職の指定(議事録 §4)
- ❌ 性格の指定(「あなたは外向的です」等)
- ❌ タスクの説明(「会議で議論してください」)

詳細は [05_プロンプト実装](../05_プロンプト実装/) 参照。

## 再現性の確保

### 固定すべきもの
- [x] `seed`(エージェントごとに一意、run ごとに固定)
- [x] `temperature`
- [x] `num_ctx` / `num_predict`
- [x] Ollama バージョン(`ollama --version` を run metadata に記録)
- [x] モデルダイジェスト(`ollama list` の SHA)

### seed の振り方

```python
# agent_seed = run_seed * 10000 + agent_id
# run 1: agent 0 → 420000, agent 1 → 420001, ...
# run 2: agent 0 → 430000, agent 1 → 430001, ...
```

これで「同一 run 内は再現性、run 間は独立」を保証。

### run metadata の保存

各実験で yaml/json で記録:

```yaml
run_id: "exp_20260425_001"
timestamp: "2026-04-25T14:00:00+09:00"
models:
  qwen3_4b:
    tag: "huihui_ai/qwen3-abliterated:4b"
    digest: "sha256:..."
env:
  ollama_version: "0.6.x"
  num_parallel: 8
  flash_attention: true
  kv_cache_type: "q8_0"
params:
  temperature: 0.8
  num_ctx: 2048
  num_predict: 256
  seed_base: 42
agents: 100
steps: 100
```

## 量子化の選択

### Q4_K_M を既定とする理由

- VRAM 効率と品質のスイートスポット
- 6GB VRAM で 3B/4B/8B いずれも現実的
- perplexity 劣化は Q8 比で極小

### 量子化別の比較

| 量子化 | サイズ(4B 基準) | 品質 | VRAM 適性 | 備考 |
| --- | --- | --- | --- | --- |
| Q3_K_M | ~2.2GB | やや劣化 | 小 | 8B を無理に積む時のみ検討 |
| **Q4_K_M** ✅ | ~2.7GB | 実用ライン | 中 | **既定** |
| Q5_K_M | ~3.0GB | 高品質 | 中-大 | 小集団品質重視で試す価値あり |
| Q8_0 | ~4.5GB | ほぼ無劣化 | 大 | 4B なら載る、3B より高品質 |
