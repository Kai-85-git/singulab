# 05_エラーハンドリング

## 1. 発生しうるエラー

| エラー | 原因 | 対処 |
| --- | --- | --- |
| `httpx.ConnectError` | Ollama サーバー未起動 | 起動確認 + 指数バックオフリトライ |
| `httpx.ReadTimeout` | 推論が遅すぎ(サーマル、モデルロード中) | timeout を 120 秒に延長、リトライ |
| HTTP 500 + "out of memory" | VRAM OOM | `num_ctx` を半減、並列度を下げる |
| HTTP 400 + "model not found" | モデル未 pull | `ollama pull` 実行後リトライ |
| HTTP 429 | レート制限(API 時) | 指数バックオフ |

## 2. リトライラッパー

```python
import asyncio
from httpx import HTTPError

async def chat_with_retry(client: AsyncLLMClient, messages, retries: int = 3, **params):
    last_exc = None
    for i in range(retries):
        try:
            return await client.chat(messages, **params)
        except HTTPError as e:
            last_exc = e
            wait = 2 ** i  # 1s, 2s, 4s
            await asyncio.sleep(wait)
    raise last_exc
```

## 3. OOM 時の段階的フォールバック

```python
FALLBACK_PARAMS = [
    {"num_ctx": 2048, "num_predict": 256},  # 既定
    {"num_ctx": 1024, "num_predict": 192},  # 1 段階目
    {"num_ctx": 512,  "num_predict": 128},  # 2 段階目(最終手段)
]

async def chat_with_fallback(client, messages, **params):
    for fallback in FALLBACK_PARAMS:
        try:
            return await client.chat(messages, **{**params, **fallback})
        except HTTPError as e:
            if "out of memory" not in str(e).lower():
                raise
    raise RuntimeError("VRAM 不足で全フォールバックが失敗")
```
