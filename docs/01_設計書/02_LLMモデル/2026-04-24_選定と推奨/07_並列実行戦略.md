# 07_並列実行戦略(RTX 3060 Laptop 6GB 向け)

## 1. 100 人大集団の実行プラン

**現実解: Llama 3.2 3B abliterated + Ollama 並列 8 + バッチ直列**

```
100 エージェント を 13 バッチ(8 × 12 + 4)に分割
→ 各バッチで同時に 8 エージェントを推論
→ 全バッチ完了で 1 ステップ終了
```

### 時間見積もり(800,000 tokens 全体)

| 構成 | スループット(集約) | 合計時間 |
| --- | --- | --- |
| 3B + Ollama 並列 8 | ~250〜350 tok/s | **~45〜55 分** |
| 3B + Ollama 並列 12 | ~350〜450 tok/s | **~30〜40 分** |
| 4B + Ollama 並列 4 | ~140〜180 tok/s | ~80〜100 分 |
| 4B + Ollama 並列 8 | ~220〜280 tok/s | ~50〜60 分 |
| 8B CPU+GPU(並列 1) | ~10 tok/s | 約 22 時間(実用外) |

→ **3B で並列 8〜12 が最もバランス良い**。ハッカソン当日に完走可能。

## 2. 小集団(2〜10 人)の実行プラン

時間に余裕があるので **品質重視 8B** が使える:
- Ollama + Dolphin 3.0 abliterated 8B(GPU+CPU オフロード)
- 並列 1、単純直列で実行
- 10 人 × 100 ステップ = 1000 呼び出し × 80 tokens = 80K tokens → **約 90 分**

## 3. vLLM / llama-server は?

6GB VRAM だと **vLLM のメリットが薄い**:
- vLLM は最低 2〜3GB の overhead がある
- モデル + KV + overhead で 6GB を超えやすい
- Ampere の FP8 非対応も相まって、Ollama と大差ない結果になりがち

→ **本機では Ollama で素直に運用するのが最適**。vLLM は「デスクトップ 3060 12GB 以上 + 大量並列」でこそ真価を発揮する。

## 4. 環境変数(Windows / Ollama)

```powershell
# システム環境変数に設定
$env:OLLAMA_NUM_PARALLEL = 8         # 3B の場合。4B なら 4-6
$env:OLLAMA_FLASH_ATTENTION = 1      # 必須
$env:OLLAMA_KV_CACHE_TYPE = "q8_0"   # KV を 8bit 量子化で節約
$env:OLLAMA_KEEP_ALIVE = "10m"       # モデル切替を減らす
$env:OLLAMA_MAX_LOADED_MODELS = 1    # メモリ圧迫を避ける
```

## 5. ノート PC 特有の注意

| 項目 | 対策 |
| --- | --- |
| Thermal throttle | 冷却台使用、ファン最大(Armoury Crate の Turbo モード)、筐体背面を浮かす |
| 電源 | **必ず AC 接続**。バッテリ駆動だと TGP が 30W に絞られ、半分以下の速度に |
| メモリ | Chrome / VS Code 等を閉じる。OS 実使用を 4GB 以下に抑える |
| 長時間稼働 | 1 時間ごとに 5 分休憩を入れるとセンサー保護が効きやすい |
