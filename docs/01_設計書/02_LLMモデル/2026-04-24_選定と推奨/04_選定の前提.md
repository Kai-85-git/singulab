# 04_選定の前提

議事録と要件定義から抽出した条件:

| 条件 | 値 | 出典 |
| --- | --- | --- |
| エージェント数 | 小集団 2〜10 + 大集団 100 | 議事録 §3.1 |
| ステップ数 | 100 ステップ | 議事録 §7 |
| 1 回の LLM 呼び出し総数 | 最大 **10,000 回** | 100 × 100 |
| 実行環境 | **RTX 3060 Laptop 6GB / 16GB RAM / Windows 11** | 本書 §01_対象ハードウェア |
| モデル方針 | **Uncensored / 脱制約系** | 議事録 §7 |
| プロンプト設計 | 短い状態記述のみ(指示過多は創発を殺す) | 議事録 §4 |
| ペルソナ | 日本人 8〜9 割 + 外国人 1 割、年齢のみ付与 | 議事録 §4 |

→ 「**超軽量 (3-4B)、日本語強い、指示追従、拒否率低め、6GB VRAM に収まる**」が選定軸。

## 除外するモデル

- **🚫 日本製モデル全般**(ELYZA / Sarashina / Swallow / PLaMo / tsuzumi / Shisa 等)
  - 公式 abliterated 版が存在せず、**Uncensored 要件**(議事録 §7)を満たさないため
  - 自前 abliteration は工数増 + 再現性リスクがあり、ハッカソン期間内では非現実的
  - 日本語品質は **Qwen3 4B abliterated で十分カバー**(Shisa.AI 自身のベンチマークで Qwen3 8B は開源 8B 級日本語性能のトップ)
- **🚫 70B+ の大型モデル**
  - 「頭の良すぎる会話」になり創発が死ぬ([02_設計哲学](02_設計哲学.md) 参照)
  - そもそも RTX 3060 6GB に載らない

## 国籍比較の新しい軸

JP 除外により比較は **🇺🇸 Llama(Meta) vs 🇨🇳 Qwen(Alibaba) の 2 極** に統一:
- 小集団・大集団ともに「同一シナリオを Llama 3.2 3B と Qwen3 4B の両方で回す」設計
- どちらも `huihui_ai` による abliteration なので **安全ガード由来の差を排除** できる(純粋な学習データ由来の思想差を観察可能)

> ⚠️ 3-4B を選ぶ理由は「VRAM 制約」だけではなく、**実人間の認知能力に近づけることがシミュレーション品質上正しい**([02_設計哲学](02_設計哲学.md) 参照)。70B 級を選ぶと全員が PhD のオフィス会議になり、創発が死ぬ。
