# 06_推奨モデルの詳細

## 1. メイン: Qwen3 4B abliterated

- **Ollama**: `huihui_ai/qwen3-abliterated:4b`
- **サイズ**: 4.02B / Q4_K_M / ~2.7GB
- **スループット(RTX 3060 Laptop 6GB)**: 単一ストリーム **35〜50 tok/s** 期待値
- **日本語**: 8B 版に次ぐ品質。実用ライン

### 選定理由
1. **RTX 3060 Laptop で最も使いやすいサイズ**(VRAM 半分を残せる)
2. **119 言語対応** → 外国人ペルソナ 1 割枠をカバー
3. abliteration で拒否率を下げつつ、Qwen3 の日本語性能を継承
4. **小集団(2〜10)と大集団(100)の両方で使える**

### 導入
```bash
ollama pull huihui_ai/qwen3-abliterated:4b
```

## 2. 100 人大集団の本命: Llama 3.2 3B abliterated

- **Ollama**: `huihui_ai/llama3.2-abliterated:3b`(※タグ名は pull 時に確認)
- **サイズ**: 3.21B / Q4_K_M / ~2.1GB
- **スループット**: 単一 **50〜70 tok/s** 期待値

### 選定理由
1. **6GB VRAM で最大並列(12)を狙える**唯一の現実解
2. Llama 3.2 系で Ollama エコシステムの実績多数
3. abliteration 済みで創発観察が可能
4. **100 人 × 100 ステップを最短時間で完走させるため**

### 導入
```bash
# 正確なタグは ollama search で確認
ollama search huihui llama3.2
ollama pull huihui_ai/llama3.2-abliterated:3b
```

## 3. (Stretch) 品質検証用: Dolphin 3.0 abliterated 8B + CPU オフロード

- **Ollama**: `huihui_ai/dolphin3-abliterated:8b-llama3.1-q4_K_M`
- **構成**: GPU に **20〜25 層だけ**オフロード、残りは CPU(Ryzen 7 6800H)
- **スループット**: **8〜15 tok/s**(単一、低速)

### 用途
- 小集団(2〜10 人)の品質検証のみ
- 100 人大集団では使わない(時間的に不可能)

### 導入
```bash
ollama pull huihui_ai/dolphin3-abliterated:8b-llama3.1-q4_K_M

# Ollama は自動で GPU/CPU 分割するが、明示制御したい場合は Modelfile で
# PARAMETER num_gpu 20  # 20 層だけ GPU に載せる
```

> 詳細な Modelfile + パラメータは [01_要件定義/02_バックエンド/02_モデル設定/03_dolphin3-8b](../../01_要件定義/02_バックエンド/02_モデル設定/03_dolphin3-8b/) を参照。
