# 08_代替案: API ハイブリッド構成

100 人大集団をローカルで回すのが時間的に厳しい、または品質に不満がある場合:

## 構成

- **小集団 + 品質検証**: ローカル(Qwen3 4B abliterated)で観察
- **大集団本番**: API 系を併用
  - **Claude Haiku 4.5**: 日本語強い、速い、価格も妥当($1/M token 台)
  - **Gemini 2.5 Flash**: 最安($0.1/M token 台)、日本語可
  - **Grok-4**: 比較的ガード弱め(議事録 §Q&A 言及)、日本語も使える

## コスト試算(800K tokens / 100 人 × 100 ステップ)

| モデル | 入力単価(USD/M) | 出力単価 | 概算総額(入出力半々) |
| --- | --- | --- | --- |
| Claude Haiku 4.5 | $1 | $5 | **約 $2.4**(約 360 円) |
| Gemini 2.5 Flash | $0.1 | $0.4 | **約 $0.2**(約 30 円) |
| GPT-4o mini | $0.15 | $0.6 | **約 $0.3**(約 45 円) |

→ **1 回の実験なら数十円〜数百円**。ハッカソン当日の本番だけ API に逃がすのは合理的。

## リスク

- **API 系は uncensored ではない**(ガードあり)→ 創発観察の趣旨とズレる
- Grok-4 のみ比較的ガードが弱い
- 提出資料での「uncensored 観察」主張と整合しなくなる

→ API は「スループット確保のバックアッププラン」として位置づけ、本線は **ローカル 3B/4B abliterated** を推奨。
