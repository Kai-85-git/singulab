# 02_設計哲学: 実人間の認知能力に合わせたモデルサイズ

## 本シミュレーションは「頭の良すぎる LLM」を**意図的に使わない**

議事録 §5.3 の 3 階層モデルは **「LLM が人間を超越しないよう、人間のキャパシティ(記憶・認知)の範囲内に収める」** を設計の核に据えている。同じ思想を **モデルサイズ選定** にも適用する。

## モデルサイズと「疑似 IQ」の対応(経験則)

| モデルサイズ | 推論特性 | 疑似的な人間 IQ | オフィス内での立ち位置 |
| --- | --- | --- | --- |
| 70B+(GPT-5 級) | 完璧に articulate、ミスしない、常に合理的 | ~140+ | **現実のオフィスには存在しない**。一人だけ PhD の天才 |
| 20-30B | 洗練された議論、論理的綻び少ない | ~125-135 | 優秀なマネージャー層 |
| **7-8B** | 普通の会話、たまに事実を取り違える | **~100-120** | **一般社員の平均像** |
| **3-4B** | シンプルな推論、繰り返しや勘違いあり | **~85-105** | **現実の職場のリアル多数派** |
| 1-2B | しばしば混乱、短い応答、反復 | ~70-85 | 疲れた日の自分 / 新人 / 眠い人 |

## なぜ 3-4B が「意思決定の創発観察」に最適か

議事録・要件定義から抽出される 5 つの根拠:

1. **兵頭氏アドバイス(議事録 §7)**: 「賢い LLM ほど『頭のいい会話』になりがち、純粋で衝動的な人間の側面が見えない」 → 3-4B は **適度な抜け・感情優位** を自然に示す
2. **役職を設定しない(議事録 §4)**: 自然発生を観察したいのに、70B は勝手に「議論のリーダー役」を完璧に演じてしまう → 小さいモデルは**役割固定化しにくい**
3. **「指示を書きすぎない」(議事録 §4)**: 大型モデルはプロンプトの微小なヒントから完璧に意図を汲み取ってしまう → 小型モデルは**解釈の揺らぎが大きく、人間的**
4. **ダンバー数・認知限界との整合**: 3 階(世界の法則)で人間の記憶制約を課すなら、モデル自体も人間に近い推論能力が自然
5. **フリーライド・諦め・無言の観察**: 賢いモデルは議論を諦めない / 無言にならない。小型モデルは **自然に「もういいや」を出力する**

## 結論

**3-4B モデルは RTX 3060 6GB への妥協ではなく、シミュレーション品質を最大化する積極的選択である。** むしろ本件に 70B を持ち込むと、全員が PhD の企業会議を演じてしまい、**創発すべき人間的ダイナミクスが死ぬ**。

## 発展: IQ の分布を持たせる(オプション)

本物のオフィスには IQ 85〜130 が混在している。エージェントごとに使うモデルサイズを振る案:

| 割合 | モデル | 役割 |
| --- | --- | --- |
| 60% | **Llama 3.2 3B abliterated** 🇺🇸 | 多数派(平均的な社員) |
| 30% | **Qwen3 4B abliterated** 🇨🇳 | やや賢い層(中堅・発言の軸になる人) |
| 10% | **Dolphin 3.0 abliterated 8B** 🇺🇸(CPU オフロード) | 「キレ者」枠(稀にしか発話しない観察者役にしても面白い) |

これにより:
- 「自然発生するリーダー」が推論能力の高いエージェントから出やすいか検証可能
- 100 人大集団でも**全員が 8B だと不自然**、だが**全員が 3B でも単調**
- 「誰が議論をリードするか」の創発が、モデルサイズ分布でどう変わるかを観察できる

→ **2 回目以降の実験**でこの IQ 分布実験を組み込むと研究価値が上がる。初回は均一モデルで OK。
