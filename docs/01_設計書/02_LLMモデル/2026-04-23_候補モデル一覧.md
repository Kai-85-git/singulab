# LLM モデル候補一覧(2026-04-23)

ハッカソン向けの LLM マルチエージェントシミュレーションで使う候補モデルを、出身国別 + Uncensored 系に整理。

**目的**:
- 同一シナリオでも **国産 LLM によって判断が変わるか** を比較(議事録 §8.1 の国際協力シミュレーション等で活用)
- 安全ガードによる創発抑制を避けるため、研究用途で **Uncensored / Abliterated** モデルも選択肢に含める

> 関連: [創発シナリオ集](../../00_アイデア/2026-04-23_創発シナリオ集.md) / [議事録 §Q&A](../../99_その他/2026-04/2026-04-22_LLMハッカソン_キックオフ議事録.md)

## 🔓 Uncensored / Abliterated モデル

> **目的**: 安全ガードが創発を潰すのを避け、**キャラ/ペルソナ指示に素直に従う** モデルで観察。研究用途限定。
>
> **注意**: ヘイト・違法行為・児童性的搾取等を生成する目的では使わない。公開提出物では出力内容を精査し、ハッカソン規約・各国法令を遵守。

### 代表シリーズ

| モデル | 特徴 | 入手 |
| --- | --- | --- |
| **Dolphin 3.0 / Dolphin 2.9** | Eric Hartford 氏製。Llama/Qwen/Mistral ベース。**定番の uncensored 系** | Ollama: `dolphin-llama3`, `dolphin-mistral`, `dolphin-mixtral` |
| **Nous Hermes 3** | Nous Research 製。指示追従とキャラ維持が優秀 | Ollama: `hermes3` |
| **OpenHermes 2.5** | Mistral 7B ベース | Ollama: `openhermes` |
| **WizardLM-2 Uncensored** | WizardLM 系の安全層除去 | HF |
| **Mlabonne's Abliterated** | **Abliteration**(重み操作で拒否方向を除去)系。Llama 3.1 / Qwen 2.5 / Gemma 2 の abliterated 版あり | HF: `mlabonne/Meta-Llama-3.1-8B-Instruct-abliterated` 等 |
| **Llama-3-Lumimaid / MythoMax** | ロールプレイ強化 | HF |
| **Solar Uncensored** | Upstage Solar 系 | HF |
| **Neural Daredevil 8B abliterated** | Llama 3 ベース abliterated | Ollama 系の GGUF |

### Abliteration とは

Llama 3 以降で広まった手法。モデル重みから **「拒否方向(refusal direction)」** を検出し、それをモデルから引き算することで、追加学習なしに安全ガードを弱める。指示追従能力は保たれやすい。

### 創発研究での使いどころ

| シナリオ | 推奨モデル |
| --- | --- |
| 嘘・裏切り・内部告発の創発 | Dolphin 3.0, Hermes 3 |
| 派閥・激しい対立・復讐の観察 | abliterated Llama 3.1 8B, MythoMax |
| 倫理ジレンマの率直な応答 | Nous Hermes 3, WizardLM-2 Uncensored |
| 非合理・感情的な意思決定 | Dolphin + 低 temperature 実験 |

```bash
ollama pull llama3.1:8b        # 基準
ollama pull qwen2.5:7b         # 中国製比較用
```

### ② 多国籍ペルソナ比較セット

```bash
ollama pull llama3.1:8b            # 米
```

```bash
# abliterated 系は HuggingFace から GGUF 取得して Ollama Modelfile で登録
```

## 実装上の注意

1. **LLM 差分評価** は、同一プロンプト × 同一環境 × 3〜10回の繰り返しで統計的に(議事録 §Q&A)
2. **temperature**: 0.7〜0.9 が創発観察のスイートスポット。再現性重視なら 0.2〜0.4
3. **コンテキスト長**: エージェント数 × 会話履歴で爆発するので、小型モデルは 8K〜32K コンテキストか確認
4. **ライセンス**: 中国系・非商用系は提出用途と公開条件を確認。提出物の README にモデル名とライセンスを明記する(議事録ルール)
5. **ローカル実行環境**:
   - 8B Q4 ≈ 5GB VRAM/RAM
   - 13B Q4 ≈ 8GB
   - 70B Q4 ≈ 40GB(ハッカソン用途では厳しい)
   - MacBook Air M1/8GB では 7B Q4 が限界(議事録 §Q&A)
