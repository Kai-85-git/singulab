# 05_3階(世界の法則) 設計

「誰も逆らえない世界のルール」。**人間のキャパシティに相当する 3 法則のみ設定**、合意形成・発言権は創発として観察する。

> 本書は [2026-04-24 ミーティング §5.3](../../02_ミーティング/2026年4月/02_議事録/2026年4月24日.md) の決定事項に基づく。
>
> ⚠️ **位置付け**: たたき段階。

## 1. 議事録からの決定

| 法則 | 設定 | 備考 |
| --- | --- | --- |
| コミュニケーション到達距離 | ✅ 設定 | 物理的・空間的制約 |
| 情報伝達の遅延 | ✅ 設定 | リアルタイム性の限界 |
| 認知限界(ダンバー数) | ✅ 設定 | 人間の記憶・認識の制約 |
| 合意形成ルール(多数決/全会一致) | ❌ 設定しない | **最も観察したい論点のため** |
| 発言権の分配 | ❌ 設定しない | 決めたらそのとおりになってしまう |

> **設計の核**: LLM が「人間を超越」しないよう、法則は人間のキャパシティ内に収める。3 法則が物理制約になり、その制約下で何が創発するかを観察する。

## 2. 設定する: 3 法則

### 2.1 コミュニケーション到達距離

#### 意義
音声・ささやきで届く物理的な範囲。

#### 定義
- エージェント i の発話は `|pos_i - pos_j| < r_comm` を満たす j にのみ届く
- place が同じでも距離が離れていれば届かない
- `r_comm` は place ごとに異なってもよい(会議室内は近距離で揃う想定)

#### 仮値

| 集団 | r_comm |
| --- | --- |
| 小集団(2〜10 人) | オフィス全体をカバー(例: 15.0 ユニット) |
| 大集団(100 人) | 近傍のみ(例: 3.0〜5.0 ユニット)。遠い人の声は届かない |

#### 実装
参考実装 [agent.py](../../10_共有資料/2d-multi-places-simulation-on-fire-public/) の `nearby_agents` 抽出ロジックに対応。

### 2.2 情報伝達の遅延

#### 意義
リアルタイム性の限界。対面 vs 伝聞の差を生む。

#### 定義
- ステップ `t` の発話は `t + d` ステップに受信される(`d ≧ 0`)
- 対面(同一 place、近距離): d = 0(即時)
- 遠隔(異なる place 間、または大集団の口コミ伝搬): d = 2〜5

#### 仮値

| 通信形態 | d |
| --- | --- |
| 同一 place、`r_comm` 以内 | 0 |
| 同一 place、`r_comm` 以遠 | 1〜2 |
| place 間の伝聞 | 3〜5 |

#### 実装
Orchestrator のメッセージ配信フェーズに **遅延キュー** を導入。`t + d` で dequeue し、対象に届ける。配送失敗(受信時点で不在等)は失われる扱い。

### 2.3 認知限界(ダンバー数)

#### 意義
一人が「覚えていられる他者」の数。

#### 定義
- 各エージェントが Memory に積める他エージェント情報を K 人に制限
- 超過時: 最後に関わってから時間の経った順に忘却(LRU)

#### 仮値

| パラメータ | 値 | 根拠 |
| --- | --- | --- |
| K(小集団) | 10 人(全員把握可能) | 小集団ではキャパを超えない |
| K(大集団) | 30〜50 人 | ダンバー数の下限(親密レベル)を目安 |

#### 実装
[06_システム設計/04_エージェント設計](../06_システム設計/04_エージェント設計/) の Memory 構造で、他者情報の保持数を K に制限。プロンプト組み立て時に K を超える古い情報は落とす。

## 3. 設定しない(= 観察する)

### 3.1 合意形成ルール
多数決 / 全会一致 / リーダー決定 / 無言で諦める — これらのどれが自然発生するかが **本シミュレーションの最大の観察点**。

### 3.2 発言権の分配
誰が発言しやすい / しにくい、が事前に決まらないようにする。ダンバー数・到達距離の物理制約のもとで、自然に「よく発言する人」が生まれるか観察する。

## 4. 実装方針

### 4.1 Config

```yaml
world_laws:
  communication_radius: 15.0  # 小集団: 15, 大集団: 4 など
  message_delay:
    same_place_near: 0
    same_place_far: 1
    cross_place: 3
  cognitive_limit: 30         # Memory に保持する他者数の上限
```

### 4.2 フック箇所

| 法則 | フック先 |
| --- | --- |
| 到達距離 | `Orchestrator.deliver_messages()` |
| 遅延 | `Orchestrator.deliver_messages()` + 遅延キュー |
| 認知限界 | `Agent.update_memory()` |

### 4.3 ログ
- 配送したメッセージの `(送信元, 宛先, 遅延, 到達/未到達)` を jsonl に残す
- 事後に「誰が誰に届いた情報のみで判断したか」を解析可能にする

## 5. 将来拡張(本フェーズ外)

- 到達距離の方向性(ささやきは狭い、大声は広い)
- 遅延のノイズ(情報が歪む・伝言ゲーム化する)
- 認知限界の個人差(年齢・属性で K を変える)
- 環境由来の法則(騒音で通信が弱まる等)
